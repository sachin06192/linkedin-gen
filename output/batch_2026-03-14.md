Before this, I would've needed to grep, trace, and manually review. Now it's one API call.

Long context isn't just about reading more. It's about reasoning across an entire system at once.

### Post 10: How I Built X

Prompt caching saved me $400 last month. Here's the 10-minute setup.

I was running the same system prompt (~4,000 tokens) on every API call. Hundreds of calls per day. That's a lot of redundant token processing.

Prompt caching lets the API skip re-reading tokens it's already seen. For my use case:

**Before:** $0.015/call average → ~$630/month
**After:** $0.007/call average → ~$230/month

Setup is dead simple:

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=[{
        "type": "text",
        "text": system_prompt,
        "cache_control": {"type": "ephemeral"}
    }],
    messages=messages
)
```

One extra field. That's it. The cache hits automatically for identical prefixes within a 5-minute window.

Three rules I follow:
1. Put the longest, most stable content first (system prompt, reference docs)
2. Keep dynamic content (user input) at the end
3. Monitor your cache hit rate — if it's below 80%, your prefix is changing too often

If you're making more than 50 API calls/day with a consistent system prompt, you're leaving money on the table.

### Post 11: Listicle / Tips

4 things the Python Optimization Ladder taught me:

**1. Profile before you optimize.** I wasted a full day optimizing a function that accounted for 0.3% of my runtime. The actual bottleneck was a database query I hadn't even looked at. `cProfile` first. Always.

**2. The biggest wins are algorithmic, not syntactic.** Switching from a nested loop to a hash lookup gave me a 200x speedup. No amount of Cython or C extensions would've gotten close. Data structures > clever code.

**3. NumPy isn't always the answer.** For small arrays (< 100 elements), pure Python loops are faster than NumPy. The overhead of array creation and type conversion kills you at small scale. Know your crossover point.

**4. async is not parallelism.** I see this confusion everywhere. `asyncio` helps with I/O-bound tasks (API calls, file reads). It does nothing for CPU-bound work. For CPU, use `multiprocessing` or `concurrent.futures`. Mixing these up is the #1 Python performance mistake I see in production code.

The best performance optimization is choosing the right tool for the right problem. Everything else is premature.

### Post 17: How I Built X

I built a dead-simple secret scanner in 15 lines of Python. Here's why and how.

After the Algolia key leak story (39 admin keys in public docs!), I wrote a pre-commit hook for my team:

```python
import re, sys

PATTERNS = [
    r"(?:api[_-]?key|secret|token|password)\s*[=:]\s*['\"][a-zA-Z0-9]{20,}",
    r"sk-[a-zA-Z0-9]{40,}",           # OpenAI/Anthropic keys
    r"ghp_[a-zA-Z0-9]{36}",           # GitHub PATs
    r"AKIA[0-9A-Z]{16}",              # AWS access keys
]

def scan(filename):
    with open(filename) as f:
        for i, line in enumerate(f, 1):
            for p in PATTERNS:
                if re.search(p, line):
                    print(f"BLOCKED: {filename}:{i} — possible secret")
                    return True
    return False

found = any(scan(f) for f in sys.argv[1:])
sys.exit(1 if found else 0)
```

Save as `.git/hooks/pre-commit`, chmod +x, done.

It's caught 3 real leaks in the first week. All from copy-pasting examples from docs without swapping in dummy values.

Is it perfect? No. Enterprise tools like GitGuardian or TruffleHog are better. But this takes 2 minutes to set up and costs $0.

The best security tool is the one you actually use.

### Post 19: Listicle / Tips

6 things about the AI agent wave that nobody's talking about:

**1. Agents fail silently.** When a chatbot gives a bad answer, you see it. When an agent takes a wrong action, you might not notice until the damage is done. Observability > capability.

**2. The "human in the loop" is becoming a bottleneck.** If your agent needs approval for every action, you've just built an expensive notification system. The real unlock is knowing which actions need approval and which don't.

**3. Memory is the unsolved problem.** Current agents forget everything between sessions. The ones that remember your preferences, past decisions, and context will feel 10x better. This is a hard technical problem that nobody's shipped well yet.

**4. Tool calling > prompt engineering.** The quality gap between a well-prompted agent and a well-tooled agent is massive. Give your agent access to the right APIs and the prompts barely matter.

**5. Multi-agent is mostly hype right now.** For 95% of use cases, one agent with good tools beats a swarm of specialized agents chatting with each other. Complexity is not a feature.

**6. The killer agent app hasn't been built yet.** We're in the "let's put AI on everything" phase. The breakout product will be something nobody's thinking about right now.
